# Recurrent failure modes

## Specification failures

- quantifier swap;
- changed normalization;
- silent continuity assumption;
- omitted degenerate case;
- proving an equivalent-looking but weaker density notion;
- adding connectedness, genericity, irrationality, or bounded degree.

## Search failures

- premature convergence on one elegant route;
- correlated agents repeating the same hidden assumption;
- declaring theorem-strength missing lemmas routine;
- spending many rounds polishing notation;
- no active counterexample lane attacking intermediate claims;
- reopening blocked routes without a new mechanism.

## Proof failures

- circular reduction;
- induction that does not reduce the parameter;
- compactness without closedness or uniform bounds;
- limit interchange without domination or uniformity;
- probabilistic independence that is not present;
- use of average behavior for a pointwise statement;
- finite computation used as universal evidence;
- formal analogy replacing proof.

## Audit failures

- discovery agents reviewing their own argument;
- majority vote used as validation;
- audit prompt reveals intended fix;
- local repair not propagated downstream;
- proof source differs from audited PDF;
- machine-check scope overstated.

## Release failures

- “solved” in title or metadata;
- missing source hash;
- no exact prior-art comparison;
- unreproducible numerical verifier;
- hidden admitted Lean statements;
- no record of refuted lemmas.
