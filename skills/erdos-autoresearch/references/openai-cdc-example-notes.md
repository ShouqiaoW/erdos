# OpenAI CDC prompt exemplar

Read `openai-cdc-example-prompt.txt` as a historically successful structural
exemplar. Preserve it verbatim for provenance; the isolated `1` and `2 OPENAI`
lines are PDF-extraction artifacts.

## Carry forward

- Assume for the research task that a complete affirmative proof exists.
- Define the exact theorem, conventions, and boundary cases.
- State precisely what a complete proof must establish.
- Enumerate attractive results that do not count.
- Use diverse, independent approaches and an explicit route registry.
- Block routes that merely rename a theorem-strength missing lemma.
- Attack every candidate with problem-specific adversarial checks.
- Require concrete lemmas, constructions, equations, or counterexamples.
- Spend at least eight hours before considering a terminal return.
- Return only a complete affirmative candidate that survives every audit and
  release gate; do not return a partial or plateau result.

## Adapt to the case

- Replace the CDC statement, definitions, insufficient results, and traps with
  the audited problem contract.
- Replace `64 concurrent agents` with the actually available safe concurrency.
- Apply the selected literature mode rather than copying the benchmark-blind
  wording mechanically.
- Use counterexamples to falsify intermediate claims and routes. Do not treat
  the affirmative-proof premise as evidence that a lemma is true.

## Non-negotiable distinction

The affirmative-proof premise is a search condition, not a proof step. It
justifies persistent proof-directed exploration; it never licenses a missing
lemma, suppressed counterexample, invalid dependency, or public “solved” claim.
